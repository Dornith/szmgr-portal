Study tools

HOW IT WORKS

The portal stores your progress and notes in your browser (localStorage) — nothing leaves your machine. Three things you can do on any page:

  • Mark progress on a section — hover any h2 heading and click the small pill on its right to cycle: ○ Not started → ◐ Studying → ⚠ Review → ✓ Mastered. Counts roll up to the dashboard above.
  • Add a note to a section — hover the heading and click the notes button next to the pill. An inline editor appears below the heading; auto-saves when you click out. A purple-bordered notes button means there's a saved note there.
  • Write a page-wide note — the floating notes button in the bottom-right of every page opens a textarea for general thoughts about the question. Press n on any page (outside an input) to toggle it.

From the page-notes panel you can Export JSON (copies all your progress + notes to clipboard for backup), Import JSON (paste a previous export to restore), or Reset (wipe everything). Data is per-origin, so progress on localhost:8765 and your-username.github.io are independent — export from one, import into the other if you switch.

How to use this portal

FROM REPO

Green-bordered blocks are directly from the szmgr repo — ported, translated, and lightly reformatted. The repo is the trusted baseline. Quoting from the repo README: "Materials were enough to pass the state exam (network + UI questions) in summer 2023."

FROM LECTURES

Teal-bordered blocks are from the merged lecture slides you provided (PDFs in materialy/). Once you send PDFs for a question, those become the source of truth for that question.

ADDED

Blue-bordered blocks are additions I made — either expansions of dense sentences, more examples (often DevOps-flavored: AWS / Terraform / Jenkins / GitLab / Java), or modern context that wasn't in the repo notes. If your exam expects only what was in lectures, you can ignore these.

INTUITION

Amber blocks distill the "how to think about this" — the mental model that makes the formal definitions click. Use these to remember concepts cold.

EXAMPLE

Purple blocks are concrete worked examples or code — SQL queries, Terraform snippets, Java code, real attack examples, etc.

WATCH

Red blocks link to short, high-quality videos (Fireship, Computerphile, ByteByteGo, Tom Scott on networks, etc.) that explain the topic intuitively. Useful when the text isn't clicking.

EXAM TIP

Yellow blocks flag things examiners ask, common gotchas, or "if you remember only one thing" points.
